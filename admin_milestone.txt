Week 1: Foundation and API Contract
Goal: make the dashboard technically ready to consume real backend data.

Day 1:

Audit every dashboard page and list the data each one needs.
Create a page-to-endpoint mapping document.
Group pages into domains: overview, users, retention, API monitoring, feedback, routes, flight search, platform health.
Day 2:

Review your existing website/backend APIs and DB models.
Identify which endpoints already exist and which must be added.
Finalize response shapes for dashboard cards, charts, tables, filters, and date ranges.
Day 3:

Set up frontend integration foundation in admin-dashboard.
Add .env support with NEXT_PUBLIC_API_BASE_URL.
Create shared API client, error handling, and common types.
Decide where to use direct backend calls vs Next proxy routes.
Day 4:

Add authentication strategy for admin dashboard.
Confirm admin token/session flow.
Protect backend endpoints for admin-only access.
Handle CORS if frontend and backend are on different origins.
Day 5:

Build and test the first full integration on the Overview page.
Replace static arrays with real API data.
Add loading, empty, and error states.
Freeze the pattern for reuse across all remaining pages.
Week 1 Deliverable

Dashboard can connect to backend.
Overview page is live with real data.
API contracts and integration pattern are finalized.
Week 2: Core Dashboard Data Integration
Goal: connect the highest-value pages that make the dashboard actually useful.

Day 6:

Connect Users, User Funnel, and Growth Metrics.
Wire filters like date range, country, device, traffic source.
Day 7:

Connect Retention and User Behavior.
Add cohort, session, and repeat-user related API responses.
Day 8:

Connect Flight Search Intelligence, Route Trends, and Redirect Analytics.
Map these to search logs, redirect events, route popularity, and conversion data.
Day 9:

Connect AI Performance and Real-Time Monitoring.
Add live or near-real-time endpoints if available.
Decide polling interval or websocket/SSE only if necessary.
Day 10:

Connect Platform Health and API Monitoring.
Pull service health, uptime, response time, p95/p99, error-rate, and incident data from backend/monitoring systems.
Week 2 Deliverable

All major business and operational dashboard pages are using real backend/API data.
Filters work consistently across pages.
Static demo values are removed from core sections.
Week 3: Feedback Modules, DB Hardening, QA, and Launch Readiness
Goal: finish remaining pages, validate DB/API correctness, and make the dashboard production-ready.

Day 11:

Connect Feedback Dashboard, Inbox, Sentiment, Issues, Live Chat, AI Insights, Heatmap.
Map them to feedback tables, support logs, chat logs, and sentiment pipelines.
Day 12:

Validate DB queries and optimize slow endpoints.
Add indexes if needed.
Add pagination, sorting, and date-filter support where tables are large.
Day 13:

Add caching and performance improvements.
Use server-side fetching where useful.
Reduce duplicate API calls and stabilize loading behavior.
Day 14:

Full QA pass across all sidebar pages.
Test auth failures, empty states, backend downtime, malformed data, and filter edge cases.
Verify mobile/tablet layout still works with live data.
Day 15:

Deployment readiness.
Configure production env vars.
Final smoke test against production/staging backend.
Prepare handoff notes: endpoints used, env setup, auth flow, known gaps.
Week 3 Deliverable

Entire dashboard is connected end to end.
DB/API queries are stable and performant.
Dashboard is ready for staging or production rollout.
